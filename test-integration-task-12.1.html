🧪 Integration Test - Task 12.1: Complete User Flow

Test Scope: This test suite covers the complete user flow for the always-on-top feature.
Requirements: 1.3, 1.4, 2.4, 3.2, 7.1
Note: Some tests require manual verification as they involve window behavior and system interactions.

Automated Tests

Manual Test Procedures

Test 1: Toggle Always-On-Top Multiple Times

Requirement: 2.4 (Immediate application of changes)

  • Open Settings and locate "Always on top" toggle
  • Open another application (browser, editor, etc.)
  • Toggle "Always on top" OFF and verify window goes behind
  • Toggle "Always on top" ON and verify window stays on top
  • Repeat toggle 3-5 times and verify consistent behavior
  • Verify no errors in console (F12)

Test 2: Setting Persistence Across Restarts

Requirement: 3.2 (Load saved preference on startup)

  • Enable "Always on top" in Settings
  • Close the application completely
  • Relaunch the application
  • Open Settings and verify toggle is still ON
  • Open another app and verify window stays on top
  • Disable "Always on top"
  • Close and relaunch application
  • Verify toggle is still OFF and window has normal z-order

Test 3: Works with Hide-From-Taskbar

Requirement: 7.1 (Simultaneous operation)

  • Enable both "Always on top" and "Hide from taskbar"
  • Verify window is NOT in taskbar
  • Open another application and verify window stays on top
  • Disable "Always on top" (keep hide-from-taskbar ON)
  • Verify window still hidden from taskbar
  • Verify window now has normal z-order
  • Enable "Always on top" again
  • Verify both features work simultaneously

Test 4: Window Stays On Top with Multiple Applications

Requirements: 1.3, 1.4 (Window remains visible above all others)

  • Enable "Always on top"
  • Open a web browser and maximize it
  • Verify DontCaught window stays visible above browser
  • Open a text editor
  • Verify DontCaught window stays visible above editor
  • Open file explorer
  • Verify DontCaught window stays visible above explorer
  • Click on each application and verify window stays on top
  • Use Alt+Tab (or Cmd+Tab) to switch apps rapidly
  • Verify window consistently stays on top

Test 5: Complete End-to-End User Flow

Requirements: All (1.3, 1.4, 2.4, 3.2, 7.1)

  • First launch: Verify "Always on top" enabled by default
  • Customize: Disable "Always on top", enable "Hide from taskbar"
  • Restart: Verify both settings persist
  • Daily use: Enable "Always on top", open multiple apps
  • Toggle during use: Disable/enable and verify immediate effect
  • Multiple sessions: Close/reopen 3 times with different settings
  • Verify no errors, crashes, or unexpected behavior
